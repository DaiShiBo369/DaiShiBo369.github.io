# Disboundia 博客项目解释与补全指南

> 审计日期：2026-06-19  
> 项目路径：`D:\Blog`  
> 当前结论：项目可以成功构建，但页面完整度主要卡在特殊页面类型、数据文件、评论/搜索配置和文章元数据上。

## 1. 项目定位

这是一个基于 Hexo 的静态博客项目，主题使用 `hexo-theme-anzhiyu`。博客名称为 `Disboundia-解缚之境`，作者为 `Disboundary`，目标域名看起来是 `disboundary.top`。

项目目前已经具备基础博客能力：

- 首页、归档、分类、标签由 Hexo 自动生成。
- 文章存放在 `source/_posts/`。
- 特殊页面由 AnZhiYu 主题的 `page.type` 分发到不同模板。
- 静态资源放在 `source/img/`、`source/json/` 和主题资源目录。
- `public/` 与 `.deploy_git/` 是构建/部署产物，不应作为主要编辑入口。

我执行过 `npm run build`，Hexo 构建成功，说明当前模板和配置没有硬性构建错误。但“构建成功”不等于“页面完整”：不少导航页面生成的是空白普通页，或只显示主题框架但没有有效内容。

## 4. Hexo 与 AnZhiYu 的渲染关系

Hexo 的基本规则：

- `source/_posts/*.md` 会变成文章。
- `source/<name>/index.md` 会变成 `/<name>/` 页面。
- `source/_data/*.yml` 会进入 `site.data`，供主题模板读取。
- `_config.anzhiyu.yml` 会覆盖 `themes/anzhiyu/_config.yml`。

AnZhiYu 的页面分发在 `themes/anzhiyu/layout/page.pug`：

| `type` 值 | 使用模板 | 需要的数据 |
|---|---|---|
| `about` | `includes/page/about.pug` | `source/_data/about.yml` |
| `album` | `includes/page/album.pug` | `source/_data/album.yml` |
| `album_detail` | `includes/page/album_detail.pug` | `source/_data/album.yml` |
| `equipment` | `includes/page/equipment.pug` | `source/_data/equipment.yml` |
| `essay` | `includes/page/essay.pug` | `source/_data/essay.yml` |
| `link` | `includes/page/flink.pug` | `source/_data/link.yml` |
| `fcircle` | `includes/page/fcircle.pug` | `_config.anzhiyu.yml` 的 `friends_vue` |
| `music` | `includes/page/music.pug` | `source/json/music.json` 或网易云歌单 |
| `tags` | `includes/page/tags.pug` | Hexo 标签 |
| `categories` | `includes/page/categories.pug` | Hexo 分类 |

关键点：只有 `index.md` 中写了正确的 `type`，主题才会加载对应的特殊页面模板。当前多个页面缺少 `type`，所以它们只是普通空页。

```yaml
  indexName:
```

```yaml
search:

  field: post
  content: true
```

## 9. 评论与留言板如何补

当前 `_config.anzhiyu.yml` 中：

```yaml
comments:
  use:
```

为空，所以文章和留言页都不会出现真正评论区。

### 9.1 推荐方案：Giscus

适合 GitHub Pages 静态博客，不需要自己部署服务。

需要准备：

1. 一个公开 GitHub 仓库。
2. 开启 Discussions。
3. 到 Giscus 官网生成 `repo`、`repo_id`、`category_id`。

配置示例：

```yaml
comments:
  use: Giscus
  text: true
  lazyload: false
  count: false
  card_post_count: false

giscus:
  repo: DaiShiBo369/DaiShiBo369.github.io
  repo_id: 填写生成值
  category_id: 填写生成值
  theme:
    light: light
    dark: dark
  option:
    data-lang: zh-CN
    data-mapping: pathname
    data-category: Announcements
    data-input-position: bottom
```

### 9.2 如果要信封留言板

项目已安装 `hexo-butterfly-envelope`，但未配置。可以在 `_config.yml` 添加：

```yaml
envelope_comment:
  enable: true
  custom_pic:
    cover:
    line:
    beforeimg:
    afterimg:
  message:
    - 有什么想问的？
    - 有什么想说的？
    - 都可以在这里留下。
  bottom: Disboundia 会认真读到每一句。
  height: 1050px
  path: comments
  front_matter:
    title: 留言板
    comments: true
```

同时仍需要配置 AnZhiYu 的评论系统，否则信封下方没有真正提交区。

## 11. 独立音乐馆 HTML 的处理

根目录有：

- `music-gallery.html`
- `music-graph.html`
- `音乐馆构建计划.md`

这套音乐馆是独立 HTML 应用，不是当前 AnZhiYu `/music/` 页的一部分。它使用内联 CSS/JS 和自己的专辑数据，功能比主题音乐页更强，包括唱片架、关系图、HTML5 播放器。

当前问题：

1. 文件在项目根目录，Hexo 默认不会发布它们。
2. 两个 HTML 文件之间用相对链接互跳，适合放在同一目录。
3. 数据写死在 HTML 内，后期维护不如 JSON/YAML 独立数据源。

可选集成方式：

方案 A：作为静态页面发布

```text
source/music-gallery.html
source/music-graph.html
```

这样 Hexo 会复制到：

```text
/music-gallery.html
/music-graph.html
```

然后菜单可增加：

```yaml
独立音乐馆: /music-gallery.html || anzhiyu-icon-compact-disc
```

方案 B：改造成 Hexo 页面

新建：

```text
source/music-gallery/index.md
```

然后把 HTML 拆成主题模板、CSS、JS、数据文件。这个工作量更大，但最利于长期维护。

方案 C：保留 AnZhiYu `/music/`，把独立音乐馆作为实验页面

这是当前最稳妥的做法。先把 `music-gallery.html` 复制到 `source/` 发布，不急着重构。

## 12. UI 能否修改底层代码

可以改，但建议分三级做。

### 12.1 第一层：只改配置，低风险

适合做：

- 菜单精简。
- 开关搜索、评论、暗色模式、PWA、统计。
- 修改首页 banner、分类入口、社交链接。
- 修改默认封面、头像、网站标题。

主要文件：

```text
_config.yml
_config.anzhiyu.yml
```

### 12.2 第二层：注入 CSS/JS，中风险但可控

当前主题已自动导入：

```text
themes/anzhiyu/source/css/_extra/anzhiyu/custom.css
```

适合做：

- 首页文章卡片更紧凑。
- 修复移动端间距。
- 调整卡片圆角、阴影、字体层级。
- 统一浅色/深色背景透明度。
- 给空页面增加提示样式。

也可以用 `_config.anzhiyu.yml` 的 `inject` 引入：

```yaml
inject:
  head:
    - <link rel="stylesheet" href="/css/custom.css" media="defer" onload="this.media='all'">
  bottom:
    - <script src="/js/custom.js"></script>
```

但更推荐直接维护主题现有的 `_extra/anzhiyu/custom.css`，因为它已经被 `index.styl` 导入。

### 12.3 第三层：改 Pug/Stylus 底层，高风险

适合做：

- 重写首页结构。
- 为 `bangumis`、`movies` 增加真正的特殊页面模板。
- 把独立音乐馆嵌入主题页面。
- 修改文章卡片 DOM 结构。
- 增加新的 `page.type`。

需要改：

```text
themes/anzhiyu/layout/page.pug
themes/anzhiyu/layout/includes/page/*.pug
themes/anzhiyu/source/css/_page/*.styl
themes/anzhiyu/source/js/*.js
```

风险：

- 主题升级时容易冲突。
- Pjax 下自定义 JS 需要处理页面切换后的重新初始化。
- 改模板后要同时检查桌面端和移动端。

如果确实要做，建议先建立一个“本项目自定义模板清单”，每次只改一个功能。

## 13. 推荐 UI 改造方向

这个博客的内容气质更像个人知识、成长记录、音乐和旅行综合站，不需要过度商业化。建议 UI 目标是：清晰、个人化、内容优先。

优先级建议：

1. 首页减少空泛特效，把最新文章、精选文章、分类入口做清楚。
2. 文章卡片补封面和 description，避免首页全靠自动摘要。
3. 特殊页面先填完整内容，再考虑复杂动画。
4. `/about/` 页保留个性，但清理示例赞赏数据。
5. `/music/` 和独立音乐馆二选一作为主入口，避免两个音乐体验割裂。
6. 主题色可以保持蓝紫，但减少整站单一蓝紫感，旅行、音乐、开发日志可以用不同封面和分类色区分。
7. 移动端优先检查导航、文章卡片、音乐播放器和关于页长卡片。

适合先做的 CSS 级改造：

- 降低首页卡片高度，让文章列表更密集。
- 给无封面文章设置统一默认封面。
- 让文章标题和摘要在移动端不溢出。
- 调整 `#page`、`#post` 透明背景，增强可读性。
- 隐藏未配置的“最新评论”模块，直到评论系统可用。

适合后做的底层改造：

- 新增 `type: movies` 页面模板，读取 `source/_data/movies.yml`。
- 新增 `type: bangumis` 页面模板，读取 `source/_data/bangumis.yml`。
- 把独立音乐馆拆成 `source/js/music-gallery.js`、`source/css/music-gallery.css`、`source/json/albums.json`。
- 给首页增加“最近项目/最近创作”模块。
